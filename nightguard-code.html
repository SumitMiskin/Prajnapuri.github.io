NightGuard — Technical Spec & Example Code

Algorithms & Data Structures (Examples)

The following code samples illustrate algorithms used in NightGuard routing, safety analysis and prioritization. These are illustrative C++ snippets and pseudocode used for teaching and prototyping — they should be adapted and hardened before production.

1. Dijkstra's Algorithm (Safest Route)

Purpose: Finds the safest route between source and destination on a graph where edge weights represent risk (lighting, crime). Time complexity: O(E log V).
Open Dijkstra Code

2. Floyd–Warshall (All-Pairs Safest Routes)

Purpose: Computes shortest/safest paths between all pairs of nodes. Useful for preprocessing alternative routes. Time complexity: O(n³).
Open Floyd–Warshall Code

3. Union–Find (Disjoint Set Union) — Connected Safe Zones

Purpose: Maintain connected components (safe zones) quickly. Used to avoid routing through isolated unsafe regions. Nearly O(1) per operation with path compression.
Open Union-Find Code

4. Quick Sort (Prioritization)

Purpose: Sort ride requests by priority (urgency, vulnerability, distance). Average time O(n log n). Fast for large datasets.
Open QuickSort Code

5. Priority Queue / Heap (Emergency Requests)

Purpose: Always extract the most urgent emergency request first. Implements O(log n) insert and extract.
Open Priority Queue Code

6. Segment Tree (Range Crime Density)

Purpose: Fast range queries (e.g., sum or max) over crime/lighting data. Query and update in O(log n).
Open Segment Tree Code

7. Fenwick Tree (Real-Time Incident Tracking)

Purpose: Lightweight structure for prefix sums and dynamic updates (e.g., real-time incident counts). Operations in O(log n).
Open Fenwick Tree Code

8. Trie & KMP (Text search and pattern detection)

Purpose: Trie for prefix-based autocomplete of locations; KMP for efficient pattern detection in user reports. Trie operations O(L), KMP O(n + m).
Open Trie & KMP Code
