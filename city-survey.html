City Survey

The population of Prajnapuri is approximately 200,000. To manage and query this large dataset efficiently we apply DAA concepts (arrays, hash tables, trees, tries, sorting, graphs).

Typical operations

  • Search by Aadhar, mobile, email, or name.
  • Insert/update/delete a record when details change.
  • Detect duplicates or inconsistent entries (e.g., same person with different mobiles).

Core data structures

  • Array: store records for sequential scans, sorting, and batch processing.
  • Hash tables (maps): O(1) exact lookup â€” hash(Aadhar) with a large prime modulus; separate maps for mobile and email for fast uniqueness checks.
  • Structured trees (BST / AVL / Redâ€‘Black): ordered indexes for name/email enabling range queries and ordered traversals.
  • Tries / prefix trees: fast prefix search for emails or mobile prefixes (useful for autocomplete and fuzzy checks).

Algorithms on top

  • Searching: hash-based O(1) lookup for exact Aadhar/mobile/email; binary search on sorted arrays or tree indexes for names; trie-based prefix search for autocomplete.
  • Sorting: quicksort/mergesort to build sorted arrays or indexes (useful for deduplication and display).
  • Use sorting + grouping as a pre-step for deduplication (group similar records together for comparison).
