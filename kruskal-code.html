Problem Statement

For large, sparsely connected regions (e.g., rural outskirts of a city), we need to design a Minimum Spanning Tree (MST) connecting all zones with minimum total cost. Kruskal's algorithm excels in sparse networks where edge weights differ significantly and we want to prioritize the shortest connections.

Solution Approach

Kruskal's algorithm sorts all edges by weight and greedily selects the shortest valid connections without creating cycles. This approach uses a union-find mechanism to track connected components.

Unlike Prim's which grows from a single starting vertex, Kruskal's works by considering all edges globally. This makes it ideal for sparse networks, distributed zone planning, and scenarios where edge weights have high variance.

5// Check if two zones belong to the same group (to avoid cycles)


6int isConnected(int zoneGroup[], int a, int b) {

10// Merge two zone groups


11void connectZones(int zoneGroup[], int n, int a, int b) {


12    int oldGroup = zoneGroup[a];


13    for (int i = 0; i < n; i++) {


14        if (zoneGroup[i] == oldGroup)


15            zoneGroup[i] = zoneGroup[b];


16    }


17}

19// Partition function for QuickSort (sorts by road cost)


20int partition(int cost[], int from[], int to[], int l, int r) {


21    int pivot = cost[l];


22    int pivotFrom = from[l];


23    int pivotTo = to[l];


24    int i = l, j = r + 1;


25


26    while (true) {


27        while (i <= r && cost[++i] <= pivot);


28        while (cost[--j] > pivot);


29        if (i >= j) break;


30        swap(cost[i], cost[j]);


31        swap(from[i], from[j]);


32        swap(to[i], to[j]);


33    }


34


35    swap(cost[l], cost[j]);


36    swap(from[l], from[j]);


37    swap(to[l], to[j]);


38    return j;


39}

41void quickSort(int cost[], int from[], int to[], int l, int r) {


42    if (l < r) {


43        int p = partition(cost, from, to, l, r);


44        quickSort(cost, from, to, l, p - 1);


45        quickSort(cost, from, to, p + 1, r);


46    }


47}
