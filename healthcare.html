Healthcare (Hospital & Patient Data)

A hospital network manages patient records, sensors, and resource scheduling, requiring efficient data structures for real-time access and updates.

Hospital & Patient Data Storage

  • Patient Records: Stored in arrays or structs. Linked Lists or Skip Lists allow quick dynamic updates.
  • Indexing: Binary Search Trees (AVL, Red-Black) index records by ID/name for fast lookup. Hash Tables map Patient IDs to records for O(1) retrieval.
  • Medical Terms: Tries store drug names and terms for quick prefix searches (auto-complete).

Graph Analysis & Epidemic Tracking

  • Contact Tracing: Patients and contacts form a graph. BFS finds nearest contacts (level-by-level), while DFS explores deep exposure paths.
  • Clustering: Union-Find groups patients by connectivity (infection clusters). BFS segmentation identifies affected wards or regions.

Time-Series & Range Queries

  • Vital Signs: Segment Trees allow O(log n) queries for range maxima (e.g., highest heart rate in the last hour).
  • Real-Time Updates: Fenwick Trees efficiently update prefix sums (e.g., total dosage given).
  • Historical Data: Sparse Tables provide constant-time O(1) range queries for fixed datasets.

Medical Record Search

  • Pattern Matching: KMP finds patterns in linear time. Rabin-Karp uses hashing for multi-pattern search (e.g., multiple diseases).
  • Large Documents: Boyer-Moore efficiently scans large notes by skipping sections on mismatches.

Scheduling, Priority & Logistics

  • Triage: Min-Heaps (Priority Queues) order patients by criticality for surgeries/treatment.
  • Appointments: Skip Lists or Sorted Lists enable O(log n) search/insertion.
  • Sorting: Merge/Quick Sort for large patient lists; Insertion/Selection Sort for small daily schedules.
  • Resource Routing: Dijkstra/Bellman-Ford model supply truck routes. Floyd-Warshall plans optimal transfer times between wards.
